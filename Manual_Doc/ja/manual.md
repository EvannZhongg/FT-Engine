# FT Engine ユーザーマニュアル

## 1. ソフトウェア概要

FT Engine は、電子審判による採点を行うためのデスクトップアプリケーションです。主な機能は次のとおりです。

1. 新しい試合を作成し、フリーモードまたはトーナメントモードを設定する。
2. Bluetooth 採点デバイスをスキャンして紐付ける。
3. 試合中に各審判の採点状況をリアルタイムで表示する。
4. オーバーレイウィンドウを使って、全画面映像や指定ウィンドウの上にスコアを重ねて表示する。
5. 試合データを保存し、終了後にレポートを確認したり、詳細ファイルを出力したりできる。

![](\images\01.png)

---

## 2. 使用前の準備

ソフトウェアを使用する前に、次の項目を確認してください。

1. PC の Bluetooth が有効で、審判デバイスを正常にスキャンできること。
2. 審判デバイスが接続可能な状態になっていること。
3. オーバーレイを使用する場合は、対象となるアプリケーションまたは試合映像を事前に開いておくこと。
4. 選手リストをインポートする場合は、`.csv`、`.xlsx`、または `.xls` ファイルを準備しておくこと。
5. 配信や画面録画を行う場合は、`OBS 保護` を有効にするかどうかを事前に確認すること。


### 2.1 ソフトウェア内の主なページ

| ページ | 用途 |
| --- | --- |
| ホーム | 新規試合の作成、履歴の確認、既存プロジェクトの続行、プロジェクトの削除 |
| トップ設定パネル | 言語切替、ショートカット設定、OBS 保護、ローカルデータ削除 |
| 試合前セットアップウィザード | プロジェクト作成、グループ設定、リストのインポート、デバイスの紐付け |
| 採点ページ | リアルタイム採点、選手切替、リセット、オーバーレイ起動、試合終了 |
| オーバーレイウィンドウ | 全画面または指定ウィンドウ上にスコアと波形チャートを表示 |
| レポートページ | 換算スコア / 生スコアの確認、CSV / ZIP / SRT 詳細のエクスポート |

## 3. トップバーとグローバル設定

オーバーレイを開いていない通常表示では、トップバーは常に表示されます。ここにはブランド表示、設定ボタン、ウィンドウ操作ボタンが含まれます。


### 3.1 トップバーでできること

1. 右上の設定ボタンをクリックすると、グローバル設定パネルが開きます。
2. 最小化ボタンをクリックすると、メインウィンドウがタスクバーに格納されます。
3. 最大化ボタンをクリックすると、ウィンドウの最大化と元のサイズへの復元を切り替えられます。
4. 閉じるボタンをクリックすると、メインウィンドウが閉じます。オーバーレイウィンドウを開いている場合は、そちらも同時に閉じます。
5. トップバーの空白部分をダブルクリックしても、最大化と復元を切り替えられます。

![](\images\02.png)

### 3.2 設定パネルでできること

#### 3.2.1 言語切替

1. 設定パネルを開きます。
2. `言語` のドロップダウンで `简体中文`、`English`、または `日本語` を選択します。
3. 言語は即時に切り替わり、ローカル設定に保存されます。

![](\images\03.png)

#### 3.2.2 リセットショートカットの設定変更

1. 設定パネルを開きます。
2. `リセットショートカット` の入力欄をクリックします。
3. 使用したいキーの組み合わせを押します。例: `Ctrl+G`、`Ctrl+1`
4. 入力したショートカットは即時に保存されます。

補足:

1. このショートカットはグローバルショートカットであり、採点ページで登録されます。
2. 採点ページで `自動` が有効な場合、このショートカットは `次へ` と同じ動作を行います。
3. 採点ページで `自動` が無効な場合、このショートカットは `ゼロ` の動作を行います。

![](\images\04.png)

#### 3.2.3 OBS 保護

1. 設定パネルを開きます。
2. `OBS 保護` で `オン` または `オフ` を選択します。

機能:

1. 有効にすると、メインウィンドウの内容は OBS にキャプチャされません。
2. オーバーレイウィンドウには影響せず、配信用オーバーレイとしてそのまま利用できます。

使用例:

1. 視聴者にはオーバーレイのスコア表示だけを見せ、操作用のメイン画面は見せたくない場合に適しています。

![](\images\05.png)

#### 3.2.4 すべてのローカルデータを削除

1. 設定パネルを開きます。
2. `すべてのローカルデータを削除` をクリックします。
3. 確認ダイアログで再度承認します。
4. この端末に保存されている試合記録、アプリ設定、ログが削除され、その後アプリは自動的に再起動します。

重要:

1. この操作は元に戻せません。
2. 削除対象には、ローカルの `app_settings.json`、`match_data` ディレクトリ、`logs` ディレクトリが含まれます。
3. 削除に失敗した場合は、アプリを終了してから再試行してください。

![](\images\06.png)

## 4. ホーム画面

ホーム画面は、アプリ起動後に最初に表示されるページです。主に新しい試合の開始と、過去プロジェクトの管理に使用します。


### 4.1 ホーム画面でできること

1. `新規試合` をクリックすると、試合前セットアップウィザードに進みます。
2. `履歴` をクリックすると、過去プロジェクトの一覧を開けます。

![](\images\07.png)

### 4.2 新しい試合を作成する

1. `新規試合` カードをクリックします。
2. ソフトウェアは現在のフロントエンドキャッシュ状態を最初にクリアします。
3. その後、試合前セットアップウィザードのステップ 1 に進みます。

適している場面:

1. まったく新しい試合を開始したいとき。
2. 古いプロジェクトのグループ、選手リスト、デバイス紐付けを引き継ぎたくないとき。


### 4.3 履歴一覧

`履歴` をクリックすると、過去プロジェクトの一覧ダイアログが表示されます。


各履歴レコードでは、次の操作が可能です。

1. `表示`
   用途: このプロジェクトのレポートページを直接開きます。
2. `続行`
   用途: このプロジェクトの設定を読み込み、試合前セットアップウィザードに戻ります。試合の続きを行ったり、デバイスを再度紐付けたりできます。
3. `プロジェクト削除`
   用途: このプロジェクトの履歴データをすべて削除します。

![](\images\08.png)

補足:

1. `続行` を選ぶと、プロジェクト名、モード、グループ、選手リスト、保存済みの審判紐付けが復元されます。
2. `プロジェクト削除` は元に戻せません。
3. レポートページからホームへ戻ると、ほかのプロジェクトも続けて確認しやすいように、履歴ダイアログが再度自動で開きます。

## 5. 試合前セットアップウィザード

試合前セットアップウィザードは、選択したモードに応じて 2 ステップまたは 3 ステップで構成されます。


### 5.1 ステップ 1: 基本設定

このページでは、プロジェクトを作成し、試合モードを選択します。

![](\images\09.png)

#### 5.1.1 できること

1. `プロジェクト名` に試合名を入力します。
2. 試合モードとして `フリーモード` または `トーナメントモード` を選択します。
3. フリーモードを選んだ場合は、`審判人数` を入力します。
4. `次へ >>` をクリックして次のステップへ進みます。
5. `キャンセル` をクリックするとホームへ戻ります。


#### 5.1.2 2 つのモードの違い

**フリーモード**

1. 複数グループを事前に設定する必要はありません。
2. システムはデフォルトグループとして `Free Mode` を自動作成します。
3. 臨時テスト、簡易採点、その場での選手追加に向いています。
4. 試合中、現在の選手が全員完了したあとでも、引き続き選手を追加できます。

**トーナメントモード**

1. 次のステップである `グループ管理` に進みます。
2. グループ構成、選手リスト、審判人数が明確な正式試合に適しています。
3. 全選手の採点が終わったあと、先頭の選手から再開できます。

#### 5.1.3 使用上の注意

1. プロジェクト名はプロジェクトディレクトリの生成に使われるため、識別しやすい名前を付けてください。
2. フリーモードでは、入力した審判人数が後続ステップで表示される紐付けカード数をそのまま決定します。
3. `履歴 -> 続行` から入った場合、このページには既存の設定が自動入力されます。

---

### 5.2 ステップ 2: グループ管理

このページはトーナメントモードでのみ表示されます。ここではグループ、審判人数、選手リストを設定します。

![](\images\10.png)

#### 5.2.1 左側グループ一覧でできること

1. 作成済みのグループをすべて確認できます。
2. 任意のグループをクリックすると、右側の編集エリアが切り替わります。
3. `+ グループ追加` をクリックすると、新しいグループを作成できます。

![](\images\11.png)

#### 5.2.2 右側設定パネルでできること

1. グループ名を編集できます。
2. そのグループの審判人数を設定できます。
3. テキストボックスに選手リストを入力できます。1 行につき 1 名です。
4. `リストをインポート` をクリックすると、Excel または CSV ファイルから名前を取り込めます。
5. グループが 2 つ以上ある場合は、現在のグループを削除できます。

![](\images\12.png)

#### 5.2.3 選手リストのインポート手順

1. `リストをインポート` ボタンをクリックします。
2. `.csv`、`.xlsx`、または `.xls` ファイルを選択します。
3. ソフトウェアは最初のワークシートを読み取り、空でない列を検出します。
4. 候補列が複数ある場合は、`インポート列を選択` ダイアログが表示されます。
5. 選手名が入っている列を選択します。
6. `インポート確定` をクリックします。
7. 取り込まれた名前は、現在のグループの選手リスト欄に追記されます。

![](\images\13.png)

#### 5.2.4 保存して次へ進む

1. `保存して次へ >>` をクリックします。
2. ソフトウェアは各グループのリストを改行ごとに分割し、正式な選手リストとして整形します。
3. その後、デバイスの紐付けページへ進みます。


#### 5.2.5 使用上の注意

1. テキストボックス内の空行は自動的に無視されます。
2. グループ名はレポートでそのまま使用されるため、分かりやすい名前にしてください。
3. インポート時、ソフトウェアは選択した列の空でない値だけを読み取ります。
4. 現在のプロジェクトを履歴から続行して開いた場合、既存のグループとリストは自動で入力されます。

---

### 5.3 ステップ 3: デバイスの紐付け

このページでは、デバイスのスキャン、審判名の設定、単一デバイス / デュアルデバイスの選択、審判スロットへのデバイス紐付けを行います。

![](\images\14.png)

#### 5.3.1 ページ上部でできること

1. 現在のスキャン状況を確認できます。
2. 検出されたデバイス数を確認できます。
3. `デバイス備考設定` をクリックして、各デバイスに備考名を付けられます。
4. `再スキャン` をクリックすると、Bluetooth デバイスを再検索できます。
5. トーナメントモードでは、開始対象のグループを切り替えられます。

![](\images\15.png)

#### 5.3.2 各審判カードでできること

1. 審判の表示名を編集できます。
2. モードとして `単一デバイス` または `デュアルデバイス` を選択できます。
3. 主デバイスを選択できます。
4. デュアルデバイスモードでは、副デバイスも選択できます。

![](\images\16.png)

補足:

1. すでに別の審判に割り当てられているデバイスは、ドロップダウンに再表示されません。
2. 単一デバイスモードへ戻した場合、副デバイスは自動的にクリアされます。
3. 単一デバイスモードでは、そのデバイスの `+`、`-`、合計スコアが表示されます。
4. デュアルデバイスモードでは、主デバイスと副デバイスの両方からデータを受信します。
5. デュアルデバイスモードでは `重大減点` が追加で表示されます。
6. レポートでの重大減点計算は、デュアルデバイス審判に対してのみ適用されます。


#### 5.3.4 デバイス備考設定

ページ上部のボタンをクリックすると、`デバイス備考設定` ダイアログが開きます。

できること:

1. スキャン済みデバイスの元の名前とアドレスを確認できます。
2. 各デバイスに任意の備考名を入力できます。
3. `保存` をクリックして確定します。

保存後の反映内容:

1. 備考名はグローバル設定に保存されます。
2. デバイス選択ドロップダウンでは、`備考名 (元の名前)` の形式で優先表示されます。
3. 備考名は現在のプロジェクト限定ではなく、アプリ全体で共通利用されます。

![](\images\17.png)

#### 5.3.5 Bluetooth デバイス名の恒久変更

デバイスの備考が変更された場合、ソフトウェアは `デバイス名を恒久変更` の確認ダイアログを表示します。

できること:

1. ハードウェア側の Bluetooth 名に書き込みたい項目を選択します。
2. `選択項目を書き込む` をクリックすると、備考名を Bluetooth 名として恒久的に書き込みます。
3. `今はしない` をクリックすると、ハードウェア名は変更せず、ローカル備考だけを保持します。

重要:

1. 恒久変更に失敗した場合でも、ローカル備考は保持されます。
2. 恒久変更できるのは、現在検出可能なデバイスのみです。


#### 5.3.6 試合開始

1. 各審判の名前、モード、主デバイス、副デバイスが正しく設定されていることを確認します。
2. `試合開始` をクリックします。
3. ソフトウェアは現在のグループ設定を保存し、デバイス接続状態の確認画面に進みます。


---

### 5.4 デバイス接続状態ダイアログ

`試合開始` をクリックすると、`接続中...` ダイアログが表示されます。

この画面では次の情報を確認できます。

1. 各審判の名前
2. 主デバイスの接続状態
3. デュアルデバイスモード時の副デバイス接続状態

表示される状態:

1. `待機中`
2. `接続中`
3. `接続済み`
4. `エラー`

#### 5.4.1 ダイアログ内でできること

1. すべてのデバイスが正常に接続されると、ソフトウェアは自動で採点ページへ進みます。
2. 8 秒以内に全デバイスが接続されない場合、`接続がタイムアウトしました。` というメッセージが表示されます。
3. `キャンセル` をクリックすると、今回の試合開始処理を中止できます。
4. `そのまま進む` をクリックすると、すぐに採点ページを開けます。

推奨事項:

1. 正式な試合では、できる限りすべてのデバイスが `接続済み` になったことを確認してから進んでください。
2. `そのまま進む` は、デバッグ、テスト、または一部デバイスを一時的に無視したい場合に向いています。

## 6. 採点ページ

採点ページは、試合中のメイン操作画面です。リアルタイムスコアの確認、選手切替、スコアのリセット、オーバーレイ起動、試合終了に使用します。

![](\images\18.png)

### 6.1 左上: 試合を終了する

1. `停止` ボタンをクリックします。
2. ソフトウェアはバックエンドへ試合停止コマンドを送信します。
3. すべての審判デバイス接続が切断されます。
4. 画面はホームへ戻ります。


補足:

1. `停止` を押しても、すでに保存された履歴データは失われません。
2. 同じプロジェクトを続けたい場合は、ホームの `履歴` から再度開いてください。

### 6.2 上中央: 現在のグループと選手の切替

中央エリアには、現在のグループと現在の選手が表示されます。

できること:

1. 現在のグループ名を確認できます。
2. 左矢印で前の選手へ移動できます。
3. 右矢印で次の選手へ移動できます。
4. ドロップダウンから任意の選手を直接選択できます。
5. 採点完了済みの選手には、ドロップダウン内でチェックマークが表示されます。

![](\images\19.png)

補足:

1. 手動で選手を切り替えると、ソフトウェアは現在の試合コンテキストを同期します。
2. 選手を切り替えた直後に、1 回リセット処理が行われます。
3. フリーモードでは、最後の選手のあとでさらに `次へ` を押すと、自動的に `Player N` が追加されます。

### 6.3 右上: 自動、オーバーレイ、次へ、ゼロ

#### 6.3.1 自動切替

1. `自動` ボタンをクリックすると、有効 / 無効を切り替えられます。

有効な場合:

1. `次へ` を押すと、現在の選手のスコアが保存され、そのまま次の選手へ切り替わります。
2. グローバルショートカットも同じ動作になります。

無効な場合:

1. `次へ` を押すと、最初に確認ダイアログが表示されます。
2. グローバルショートカットは `ゼロ` の動作になります。


#### 6.3.2 オーバーレイウィンドウを起動する

1. `オーバーレイ` ボタンをクリックします。
2. ポップアップで対象ウィンドウ、または `フルスクリーンモード` を選択します。
3. `オーバーレイ開始` をクリックします。
4. 独立した透過オーバーレイウィンドウが開きます。

![](\images\20.png)

補足:

1. ソフトウェアは現在のシステム上にあるウィンドウタイトルを読み取ります。
2. 特定ウィンドウを選択した場合、オーバーレイはその時点の位置とサイズに合わせて開きます。
3. 現在の実装では、起動後にオーバーレイが対象ウィンドウの移動を継続追従することはありません。対象ウィンドウが動いた場合は、オーバーレイを手動で移動してください。

#### 6.3.3 次へ

1. `次へ` ボタンをクリックします。
2. 現在の選手には `採点完了` の状態が付きます。
3. ソフトウェアは現在のグループ内で、まだ完了していない次の選手を検索します。
4. 見つかった場合、その選手へ切り替えたうえで自動的にリセットします。

全選手の採点が完了した後:

1. フリーモードでは、新しい選手が自動で追加されます。
2. トーナメントモードでは、`全員の採点が完了！` ダイアログが表示されます。


#### 6.3.4 ゼロ

1. `ゼロ` ボタンをクリックします。
2. 現在のすべての審判スコアが 0 にリセットされます。

確認スキップが有効でない場合:

1. 確認ダイアログが表示されます。
2. `次回から確認しない` をチェックできます。


### 6.4 審判スコアカード

中央エリアの各カードは 1 名の審判を表します。

各カードに表示される内容:

1. 審判名
2. 主デバイスの接続状態インジケーター
3. デュアルデバイスモード時の副デバイス接続インジケーター
4. 現在の合計スコア
5. 現在の `+` スコアと `-` スコア
6. デュアルデバイスモード時の `重大減点`

![](\images\21.png)

インジケーター色の目安:

1. 緑: 接続済み
2. それ以外の色: 待機中、接続中、またはエラー

### 6.5 全員完了ダイアログ

現在のグループの全選手が完了すると、`全員の採点が完了！` ダイアログが表示されます。

![](\images\22.png)

利用できる操作:

1. `保存して終了`
   用途: 試合を終了してホームへ戻ります。
2. `続行`
   フリーモード: 続行し、新しい選手を追加します。
   トーナメントモード: 先頭の選手から再度採点を始めます。

補足:

1. トーナメントモードの `続行` は、グループ全体を再採点したい場合に向いています。
2. フリーモードの `続行` は、その場で選手を追加したいときに便利です。

## 7. オーバーレイウィンドウ

オーバーレイウィンドウは、試合映像の上に採点情報を重ねて表示するための独立ウィンドウです。OBS、画面投影、会場スクリーンなどでの利用に適しています。

![](\images\23.png)

### 7.1 基本的な特徴

1. デフォルトで常に最前面に表示されます。
2. デフォルトではタスクバーアイコンを表示しません。
3. デフォルトではマウスクリックが透過され、対象画面の操作を妨げません。
4. マウスカーソルが上部ドック領域またはカード上に来ると、自動的に操作可能モードへ切り替わります。

### 7.2 上部ドックバーでできること

マウスをオーバーレイ上端へ移動すると、操作用ドックバーが表示されます。

![](\images\24.png)

利用できる操作:

1. 現在の選手名を確認できます。
2. `次の選手` をクリックすると、次の選手へ切り替わります。
3. `R` をクリックすると、現在のスコアをリセットできます。
4. 波形ボタンをクリックすると、波形チャートの表示 / 非表示を切り替えられます。
5. 設定ボタンをクリックすると、オーバーレイ設定パネルを開けます。
6. 閉じるボタンをクリックすると、オーバーレイウィンドウを閉じます。

補足:

1. `次の選手` を押した場合も、現在の選手は採点完了として扱われます。
2. ドックバーは左右へドラッグできますが、Y 軸位置は上端に固定されています。

### 7.3 審判カードでできること

各審判カードは個別にドラッグできます。


さらに、次の操作も可能です。

1. 右下のリサイズハンドルをドラッグして、審判カードのサイズを変更できます。
2. 1 枚の審判カードをリサイズすると、すべての審判カードに同じサイズが適用されます。
3. スコア文字の上でマウスを押したままホイールを回すと、すべての審判カードの文字サイズをまとめて拡大 / 縮小できます。


### 7.4 波形チャート

オーバーレイウィンドウは、波形チャートコンポーネントの表示に対応しています。


波形チャートの特徴:

1. 現在の選手に初めて 0 以外のスコアが入った時点で記録を開始します。
2. X 軸は時間、Y 軸はスコアです。
3. 審判 1 名につき 1 本のラインが表示されます。
4. 選手を切り替えると、波形チャートは自動でリセットされます。
5. すべての審判スコアを 0 に戻した場合も、波形チャートはリセットされます。
6. 波形チャートカード自体もドラッグ可能で、自由にリサイズできます。

### 7.5 オーバーレイ設定パネル

ドックバーの設定ボタンをクリックすると、設定パネルが開きます。


設定できる項目:

1. `表示モード`
   選択肢:
   `分割（+ / -）`、`合計のみ`、`合計 + 分割`、`リアルタイム（バースト）`
2. `不透明度`
   スライダーで背景の透過度を調整します。
3. `背景色`
   カード背景色をカスタマイズできます。
4. `デフォルトに戻す`
   オーバーレイの見た目を初期状態へ戻します。
5. `閉じる`
   設定パネルを折りたたみます。


4 つの表示モードの説明:

1. `分割（+ / -）`: 加点と減点を分けて表示します。
2. `合計のみ`: 合計スコアだけを表示します。大型スクリーンでの表示に向いています。
3. `合計 + 分割`: 合計スコアと簡略化した内訳を同時に表示します。
4. `リアルタイム（バースト）`: 短時間の累積スコア変化を表示します。動的字幕や即時フィードバックに向いています。

補足:

1. オーバーレイの見た目設定はローカルに保存され、次回起動時にも再利用されます。
2. 現在の実装では、文字サイズの調整は全体共通であり、審判カードごとに個別設定はできません。

## 8. レポートページ

レポートページでは、過去プロジェクトの結果確認、換算スコアの計算、生スコアの表示、CSV・ログ・字幕ファイルのエクスポートを行えます。

![](\images\25.png)

### 8.1 左側グループサイドバー

左側には、現在のプロジェクトに含まれるすべてのグループが表示されます。

利用できる操作:

1. 任意のグループをクリックして、現在のレポート内容を切り替えます。
2. `← 一覧へ戻る` をクリックすると、ホームの履歴一覧状態へ戻ります。

> 画像プレースホルダー: レポートページのグループ一覧

### 8.2 上部ツールバー

上部ツールバーでは、現在の表示内容に応じてさまざまな操作を行えます。


利用できる操作:

1. `詳細設定`
   `換算スコア` 表示のときだけ表示されます。
2. `詳細をエクスポート`
   選手ごとのログや字幕ファイルを ZIP で出力します。
3. `CSV テーブル`
   現在の表ビューを CSV として出力します。
4. `換算スコア / 生スコア`
   レポート表示モードを切り替えます。

### 8.3 換算スコア表示

換算スコア表示では、次の内容を確認できます。

1. 順位
2. 選手名
3. 各審判の換算スコア
4. `重大減点` 列
5. 最終スコア

#### 8.3.1 換算ルール

現在のコードロジックに基づく換算方法は次のとおりです。

1. 現在のグループにおける、各審判の最高生スコアを求めます。
2. 各選手・各審判のスコアを、次の式で換算します。
   `player_raw_score / judge_max_score x ratio`
3. デフォルトの比率は `60` です。
4. 最終スコアは、全審判の換算スコアの平均値です。
5. `重大減点を表示` が有効な場合、最終スコアから標準重大減点がさらに差し引かれます。

#### 8.3.2 重大減点ルール

現在の実装では、次のルールで処理されます。

1. 重大減点は、デュアルデバイス審判の値のみが集計対象です。
2. 採用値は `多数決優先` で決まります。
3. 同じ票数の値が複数ある場合は、より大きい値が採用されます。

### 8.4 生スコア表示

生スコア表示では、次の内容を確認できます。

1. 各選手の、各審判における合計スコア
2. 対応する `+` 値と `-` 値
3. 重大減点がある場合は、その値もセル内に表示されます。
4. 選手の生スコア平均

![](\images\26.png)

### 8.5 詳細設定

`換算スコア` 表示で `詳細設定` をクリックすると、詳細設定ダイアログが開きます。

> 画像プレースホルダー: 詳細設定ダイアログ

利用できる操作:

1. `比率 (%)` を編集できます。
2. `重大減点を表示` の有効 / 無効を切り替えられます。
3. `確認` をクリックすると、現在の表示設定が保存されます。

補足:

1. `重大減点を表示` の状態は、プロジェクトごとのローカル設定として保存されます。
2. 同じプロジェクトのレポートを再度開いたときは、前回の重大減点表示状態が自動で復元されます。

### 8.6 詳細をエクスポート

`詳細をエクスポート` をクリックすると、`スコア詳細のエクスポート` ダイアログが開きます。


利用できる操作:

1. エクスポートしたい選手を選択できます。
2. `すべて` を使って全選手をまとめて選択できます。
3. 出力形式として次を選べます。
   `CSV（ログ）`、`SRT（字幕）`
4. `SRT` を選択した場合は、さらに SRT モードを選択します。
   `合計スコア`、`加点 / 減点`、`リアルタイムバースト`
5. `ZIP をダウンロード` をクリックします。

エクスポート結果の補足:

1. ソフトウェアは `Details_グループ名.zip` という名前の ZIP パッケージをダウンロードします。
2. ZIP 内のファイルは `グループ / 選手 / 審判` の階層で整理されます。
3. `CSV（ログ）` には時系列ログが含まれます。
4. `SRT` ファイルは、字幕オーバーレイや後編集用途に適しています。

### 8.7 テーブル CSV をエクスポート

`CSV テーブル` をクリックすると、現在の表示に対応した CSV が直接ダウンロードされます。

ルール:

1. `換算スコア` 表示では、順位、換算スコア、重大減点、最終スコアが出力されます。
2. `生スコア` 表示では、選手データ、各審判の生スコア詳細、平均スコアが出力されます。

## 9. 過去プロジェクトの続行・表示・削除

この章はホームの `履歴` 一覧を補足するもので、日常的なプロジェクト管理に役立ちます。

### 9.1 プロジェクトを続行する

操作手順:

1. ホーム
2. `履歴`
3. 対象プロジェクトを探す
4. `続行` をクリックする

続行後の動作:

1. ソフトウェアはこのプロジェクトの `config.json` を読み込みます。
2. 試合前セットアップウィザードに入ります。
3. 既存のグループ、選手リスト、審判人数、紐付け設定が復元されます。
4. 設定を調整したうえで、再び試合を開始できます。

### 9.2 レポートを表示する

操作手順:

1. ホーム
2. `履歴`
3. 対象プロジェクトを探す
4. `表示` をクリックする

すると、そのプロジェクトのレポートページが直接開きます。

### 9.3 プロジェクトを削除する

操作手順:

1. ホーム
2. `履歴`
3. 対象プロジェクトを探す
4. ゴミ箱ボタン、または `プロジェクト削除` をクリックする
5. 確認ダイアログで削除を確定する

削除後の動作:

1. プロジェクトディレクトリは完全に削除されます。
2. そのプロジェクト用のローカルレポート設定も削除されます。

## 10. ファイルとデータ

データ保存先を把握しやすいように、この章では保存内容と保存場所を簡潔に説明します。

### 10.1 プロジェクトデータに含まれる内容

各プロジェクトには、次の内容が保存されます。

1. プロジェクト設定ファイル `config.json`
2. 各グループのディレクトリ
3. 各選手・各審判ごとの生 CSV 記録ファイル

一般的な構成例:

```text
match_data/
  20260320_120000_My Match/
    config.json
    Group A/
      Player 1_Ref1.csv
      Player 1_Ref2.csv
      Player 2_Ref1.csv
```

### 10.2 設定ファイルに保存される内容

グローバル設定には、次の項目が保存されます。

1. 言語
2. ショートカットキー
3. 確認ダイアログをスキップするかどうか
4. デバイス備考
5. OBS 保護のオン / オフ
6. プロジェクトごとのレポート表示設定

### 10.3 データの保存場所

現在のコードロジックでは、保存場所は次のとおりです。

**開発環境**

1. データはデフォルトでプロジェクトルートに保存されます。
2. たとえば、現在のプロジェクト内の `app_settings.json` や `match_data` です。

**パッケージ版の本番環境**

1. Windows では通常 `%APPDATA%/FT Engine/`
2. macOS では通常 `~/Library/Application Support/FT Engine/`

### 10.4 エクスポートファイルの種類

| エクスポート種別 | 内容 |
| --- | --- |
| レポート CSV | 現在のテーブル表示内容 |
| ZIP 詳細パッケージ | 選手ログ CSV と SRT 字幕ファイル |
| ログ CSV | 時系列のスコア変化 |
| SRT 合計スコア | 合計スコアの変化 |
| SRT 分割表示 | `+ / -` の変化 |
| SRT リアルタイムバースト | 短時間の累積変化 |

## 11. 初めて使う方向けの推奨フロー

初めてこのソフトウェアを使う場合は、次の流れで進めるのがおすすめです。

### 11.1 クイックスタート手順

1. ソフトウェアを起動します。
2. 上部設定で、言語、ショートカット、OBS 保護の必要有無を確認します。
3. ホームへ戻り、`新規試合` をクリックします。
4. 基本設定で試合モードを選択します。
5. トーナメントモードを使う場合は、グループと選手リストの設定を完了します。
6. デバイスの紐付け画面で、デバイスをスキャンして割り当てます。
7. 各審判に分かりやすい名前を設定します。
8. 試合を開始し、デバイス接続が完了するのを待ちます。
9. 採点ページで採点と選手切替を進めます。
10. ライブ表示が必要な場合は、オーバーレイウィンドウを開きます。
11. 試合終了後は、`履歴` からレポートを開いて内容確認やデータ出力を行います。

## 12. 使用上の詳細と注意点

### 12.1 Bluetooth とデバイス備考

1. スキャン時に `Bluetooth が無効か利用できません。` と表示された場合は、まず PC 側の Bluetooth を確認してください。
2. スキャン結果は RSSI 順で並ぶため、信号が強いデバイスほど上に表示されます。
3. デバイス備考はプロジェクト単位ではなく、グローバルに保存されます。
4. 恒久的なデバイス名変更は任意機能であり、必須ではありません。

### 12.2 試合進行に関する注意

1. 選手を切り替えると、ソフトウェアは試合コンテキストを同期したうえで現在のスコアをリセットします。
2. フリーモードは現場での柔軟な運用に向いており、トーナメントモードは正式な試合管理に向いています。
3. 採点ページの `自動` スイッチはショートカット動作に影響するため、実運用では特に重要なポイントです。

### 12.3 オーバーレイに関する注意

1. オーバーレイはデフォルトでクリック透過になっており、元ウィンドウの操作を妨げません。
2. ドラッグや設定変更ができるのは、マウスが上部ドック領域またはカード領域にあるときだけです。
3. 審判カードのサイズは連動しており、1 枚を変更すると全カードへ反映されます。
4. 波形チャートは独立したカードとして扱われ、個別にドラッグやリサイズができます。

### 12.4 データ安全性に関する注意

1. プロジェクト削除では、そのプロジェクトのデータだけが削除されます。
2. ローカルデータ削除では、すべてのプロジェクト、すべての設定、すべてのログが削除されます。
3. 試合終了後に整理やエクスポートを進める前に、`履歴` でプロジェクトデータが正しく作成されていることを確認してください。
